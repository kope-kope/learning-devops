## üë• Users and Groups

### Core Concepts
- **Users and groups** control access to files and system resources
- Users authenticate by providing username and password
- Every file is owned by a user and associated with a group
- Every process has an owner and group affiliation, determining resource access
- Each user is assigned a unique **User ID (UID)**

### Storage Locations
- **Usernames and UIDs**: `/etc/passwd`
- **Encrypted passwords**: `/etc/shadow`
- **Group information**: `/etc/group`

### User Types

1. **Root User**
   - UID: 0
   - Home directory: `/root`
   - Default shell: `/bin/bash`
   - Full system privileges

2. **Regular Users**
   - UID range: 1000-60000
   - Maximum ~59,000 regular users can be created
   - Standard user accounts for system access

3. **Service Users**
   - System accounts for running services
   - Limited privileges for security

### User Management Operations
- Create new users
- Set and modify passwords
- Switch between user shells
- Add users to groups
- Manage user permissions
